- docker commands
	- docker ps -> lists all the docker images
	- docker run image-name -> run the image
	- Restart the container:
		- stop the container -> docker stop <id of the container>
		- if you want to start it again -> docker start <id of the container>
	- start the container and get the id -> docker run -d image-name
	- docker ps -a -> lists all the running and stopped containers (This helps when you restart the laptop next day and want to start the same container, we can get the container id from the list and start the same container again)
	- port binding -> docker run -p6000:6379 image-name (so here, the 6000 port of laptop is bind with the 6379 port of the container)
	- docker logs <container id/ container name>
	- docker run -d -> -d to run it in detach mode
	- docker run -d -p6001:6379 --name redis-older redis:4.0 (The --name flag is used to name the container, here we named it redis older cause it uses redis 4.0)
	- docker exec -it container-id  /bin/bash -> (-it stands for interactive terminal)
	- docker network ls -> lists all docker networks
	- docker network create network-name
